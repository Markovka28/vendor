                    "Команды подключения к удалённым репозиториям"
1) [Создать репозиторий] на GitHub
2) [Создать папку с проектом] на локальном компьютере или войти в нее.
    Создадим папку с именем: <Get> 
3) Войти в терминал и перейти в папку с проектом: [cd "C:\Users\Валерий Крылов\Desktop\GB\СПЕЦИАЛИЗАЦИЯ DevOps\9. Контроль версий углублённо (GIT)\Урок 1. Работа с удалёнными репозиториями"]
4) [git init] : инициализируем в этой папке репозиторий
5) [git add .\Get.html] : добавить файл в удаленный репозиторий
6) [git commit -m 'initial Get' ] : составить коммит для отправки в удаленный репозиторий
7) [git remote add origin git@github.com:Markovka28/GetWork.git] 
    подключить репозиторий к удаленному
8) [git push -u origin master] : отправить локальный репозиторий в удаленный репозиторий
    * [-u] : текущую ветку связать с удаленной
9) обновить страницу с удаленным репозиторием

10) [git clone git@github.com:Markovka28/GetWork.git] : клонировать удаленный репозиторий в папку с локальным
11) [cd GetWork/] : перейти в папку с локальным репозиторием
12) создать новый файл
13) [git add .] добавить 2й файл в удаленный репозиторий
14) [git commit -m 'two_file add'] добавить коммит
14) [git checkout -b 'two_some'] создать новую ветку
15) добавить изменения в созданный файл 

16) [git add .] зафиксировать изменения
17) [git commit -m 'changes'] зафиксировать изменения

18) [git push -u origin two_some] отправить изменения в удаленный репозиторий
    с имененм созданной ветки

19) [git remote show origin] посмотреть какие локальные ветки связанны с ветками 
    удаленного репозитория

производим изменения на удаленном репозитории

[git fetch] скачать в локальный репозиторий данные из удаленного без (слияний) обновления ветки на которой находимся

[git log origin/two_some ^two_some] посмотреть какие именно изменения внесены в удаленной ветки

[git diff fb434217c4] посмотреть какие именно изменения внесены

[git pull ] влить изменения из удаленного репозитория в локальный

[git remote -v] информация о путях к удаленному репозиторию
                для Push и fetch

                        "Заменить репозиторий origin"
- сначала удаляем за тем меняем путь к этому репозиторию

[git remote remove origin] удалить текущий репозиторий
[git remote -v] ничего не покажет
[git remote add origin https://Krylov.ru] снова добавить удаленный репозиторий
    * <https://Krylov.ru> путь миожет быть любой

[git remote set-url origin git@github.com:Markovka28/GetWork.git] 
    * восстановить путь к удаленному репозиторию

[git remote set-url --push https://путь] Поменять только путь к push

        "добавить в локальном репозитоири несколько уделанных"

[git remote add vendor git@github.com:Markovka28/vendor.git]
[git remote -v] покажет что в локальном репозитории добавлен еще один
